Create `src-tauri/src/auto_applier/executor.rs` with these components:

1. **ClickStep struct** — represents one click action in the sequence:
   ```rust
   #[derive(Debug, Clone)]
   pub struct ClickStep {
       pub x: u32,        // 1080p reference x
       pub y: u32,        // 1080p reference y
       pub label: String,  // Human-readable description for progress events
   }
   ```

2. **build_step_sequence(variant: &Variant, res: &Resolution) -> Vec<ClickStep>** — pure function that converts a Variant into ordered click steps:
   - First: iterate `variant.skill_order` (or `variant.skill.keys()` if skill_order empty). For each skill_id with rank > 0, generate `rank` number of ClickSteps at the skill allocate button position (using SkillTreeCoords::ALLOCATE_BUTTON as base, with skill-specific offset derived from index * SKILL_GRID_SPACING). Label: `"Skill {skill_id} point {n}/{rank}"`.
   - Then: for each equip_skill in `variant.equip_skills`, generate one ClickStep at the equip slot position. Label: `"Equip skill {key}"`.
   - Then: for each paragon board in `variant.paragon` (sorted by index): generate one ClickStep to navigate to that board (using BOARD_NAV_NEXT if index > 0), then one ClickStep per node in `board.nodes` at a grid position offset from ParagonBoardCoords::CENTER. Label: `"Paragon {board_name} node {node_id}"`.
   - All coordinates are 1080p reference values; scale_coord is applied later in the click function.

3. **click_at(x: u32, y: u32) -> Result<(), ApplyError>** — async function, wraps enigo in spawn_blocking:
   ```rust
   #[cfg(windows)]
   pub async fn click_at(x: u32, y: u32) -> Result<(), ApplyError> {
       tokio::task::spawn_blocking(move || {
           use enigo::{Button, Coordinate, Direction::Click, Enigo, Mouse, Settings};
           let mut enigo = Enigo::new(&Settings::default())
               .map_err(|e| ApplyError::InputFailed(e.to_string()))?;
           enigo.move_mouse(x as i32, y as i32, Coordinate::Abs)
               .map_err(|e| ApplyError::InputFailed(e.to_string()))?;
           enigo.button(Button::Left, Click)
               .map_err(|e| ApplyError::InputFailed(e.to_string()))?;
           Ok::<_, ApplyError>(())
       }).await.map_err(|e| ApplyError::TaskPanic(e.to_string()))?
   }

   #[cfg(not(windows))]
   pub async fn click_at(_x: u32, _y: u32) -> Result<(), ApplyError> {
       Err(ApplyError::InputFailed("Mouse input only available on Windows".to_string()))
   }
   ```

4. **bring_window_foreground(hwnd)** — cfg(windows) function that calls SetForegroundWindow before automation starts. Non-windows stub returns Ok.

5. **run(plan: BuildPlan, app: tauri::AppHandle, state: Arc<Mutex<AppState>>) -> Result<(), ApplyError>** — the main executor:
   - Extract game_state (resolution, hwnd) and cancel_flag from AppState under brief lock, then DROP the lock
   - Verify resolution is Some, else return ApplyError::UnsupportedResolution
   - Call bring_window_foreground(hwnd)
   - Take first variant: `plan.variants.first()` (per Phase 3 decision: first variant auto-selected)
   - Call build_step_sequence(variant, &resolution) to get ordered ClickSteps
   - Set apply_phase to Running { step: 0, total: steps.len() } under brief lock
   - Emit SafetyEvent::AutomationStarted via `app.emit("safety_event", ...)`
   - Check if resuming from a paused step (read apply_phase; if Paused { step, .. }, start from that step index)
   - For each step (starting from resume_index):
     a. Check cancel_flag — if set, check apply_phase: if Paused return Ok (pause is not error), else return ApplyError::Cancelled
     b. Capture screenshot via game_capture::screenshot::capture_window (cfg(windows))
     c. Call safety::assert_safe_state with pixels, width, height, cancel_flag
     d. On safety error: emit SafetyEvent::AutomationAborted, set apply_phase to Aborted, return error
     e. Scale coordinates: `let (sx, sy) = coords::scale_coord(step.x, step.y, &resolution)`
     f. Apply jitter: `let (jx, jy) = humanize::jitter_coord(sx, sy)`
     g. Call click_at(jx, jy).await
     h. Update apply_phase to Running { step: i+1, total } under brief lock
     i. Emit progress: `app.emit("apply_progress", serde_json::json!({"step": i+1, "total": total, "label": step.label}))`
     j. Sleep for humanize::random_delay_ms()
   - After loop completes: set apply_phase to Complete, emit "apply_complete" event
   - Return Ok(())

6. **pause(state: &Mutex<AppState>)** — sets cancel_flag to true, updates apply_phase from Running{step,total} to Paused{step,total}

7. **resume(app: AppHandle, state: Arc<Mutex<AppState>>)** — clears cancel_flag, spawns a new tokio task that:
   - Reads build_plan from state (clone)
   - Calls run() which will detect Paused state and resume from saved step
