## Components

### Component Inventory (Phase 3 GUI surface)

Phase 3 introduces exactly two interactive surfaces:

**1. LinkInput** — Build link paste field
- Single `<input type="text">` accepting full d2core.com URL
- Placeholder: `粘贴 d2core.com 构建链接` (source: CONTEXT.md Chinese-only UX decision)
- Full-width, single row, no multi-line
- Right-aligned submit button inline or below: `解析构建`
- Input background: `#262626`, border: `1px solid #404040`, border-radius: `4px`
- Input padding: 12px horizontal (md − 4px for visual balance), 8px vertical (sm)
- On focus: border color shifts to `#d4af37` (accent)
- On submit with empty value: show inline validation `请粘贴构建链接` below input

**2. BuildPreview** — Read-only parsed build display card
- Visible only after successful parse; absent on first load (empty state applies)
- Card background: `#262626`, border: `1px solid #404040`, border-radius: `8px`
- Card internal padding: 24px (lg)
- Structure:
  - Row 1: Build title (Heading, 20px/600) — raw API `data.title` value
  - Row 2: Class label (Label, 12px/600, text-secondary) + class value (Body, 14px/400)
  - Divider: 1px `#404040` line, 16px vertical margin
  - Section header "技能" (Label, 12px/600, text-secondary)
  - Skill list: up to 6 rows — skill key (Body, 14px/400) + rank chip (12px, accent color)
    - Skill keys shown as raw API values (e.g. `druid_wolves`) — name mapping deferred to Phase 5
  - Divider: 1px `#404040` line, 16px vertical margin
  - Section header "传奇天赋" (Label, 12px/600, text-secondary)
  - Paragon list: board name (Body, 14px/400) + glyph if present (Body, 14px/400, text-secondary)
    - Boards displayed in `index` order (ascending)

### Loading State
- Displayed between submit and API response
- Text only: `正在解析...` (Body, 14px/400, text-secondary), centered below input
- No spinner required in Phase 3 — API calls are ~50ms, a text indicator is sufficient

## Interaction Contract

### Parse flow
1. User pastes URL into LinkInput
2. User clicks "解析构建" (or presses Enter)
3. App validates URL contains `d2core.com` and `bd=` parameter — if not, show inline validation error immediately (no API call)
4. Loading text "正在解析..." appears below input
5. API call completes (~50ms):
   - **Success**: BuildPreview card renders below loading text; loading text disappears
   - **Failure**: Error message replaces loading text (specific copy per error type above)
6. Subsequent parses: BuildPreview card updates in place (no animation required)

### Error display placement
- API errors: displayed as a text block below the input, same position as loading text
- Color: `#dc2626` (destructive)
- Size: Body 14px/400

### Keyboard
- `Enter` in input field triggers parse (same as clicking CTA button)
- No other keyboard shortcuts in Phase 3
